# `apps/website`

The site you are reading. Docusaurus 3.9 with a custom sidebar and a live playground that mounts the real resolver in the browser.

## Layout

```text
apps/website/
├── docs/              # .mdx docs (this site)
├── src/
│   ├── pages/
│   │   ├── index.tsx       # homepage
│   │   └── playground.tsx  # /playground route
│   ├── components/
│   └── css/custom.css      # Infima overrides + site styles
├── static/img/       # logo, theme screenshots, demo gif
├── docusaurus.config.ts
├── sidebars.ts
└── package.json
```

## Playground

`src/pages/playground.tsx` lazy-imports the real resolver and a demo conflict:

```tsx
const [
  { ConflictResolver },
  { matchCells, analyzeSemanticConflictsFromMappings },
  { injectStyles },
  baseNb,
  currentNb,
  incomingNb,
] = await Promise.all([
  import('../../../../packages/web/client/src/components/ConflictResolver'),
  import('../../../../packages/core/src'),
  import('../../../../packages/web/client/src/styles'),
  import('../../../../test-fixtures/demo/base.ipynb'),
  import('../../../../test-fixtures/demo/current.ipynb'),
  import('../../../../test-fixtures/demo/incoming.ipynb'),
]);
```

The `.ipynb` imports work because of a tiny `ipynb-loader` webpack plugin wired up in `docusaurus.config.ts` that registers `.ipynb` as a JSON module.

`onResolve` and `onCancel` are no-ops here: no server, no Git, no disk writes. It is strictly a UI sandbox.

## Build and develop

```bash
cd apps/website
npm install
npm run start    # http://localhost:3000
npm run build    # production build into build/
npm run serve    # serve the production build locally
```

The site is deployed to GitHub Pages at `https://Avni2000.github.io/MergeNB/` from the `gh-pages` branch.
