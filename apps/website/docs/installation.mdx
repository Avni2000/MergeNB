# Installation

MergeNB ships as a VS Code extension. Two paths: grab a release, or build from source.

## From a release (recommended)

1. Download the latest `.vsix` from the [Releases page](https://github.com/Avni2000/MergeNB/releases).
2. Open VS Code.
3. Open the Command Palette and run **Extensions: Install from VSIX...**
4. Select the downloaded file.

:::info VS Code Marketplace
A Marketplace listing is in the works. Until then, VSIX is the shipping channel.
:::

## Requirements

| Requirement | Version |
|---|---|
| VS Code | `^1.80.0` |
| Node.js (build from source only) | `>= 20` |
| Git | any recent version |

The extension declares `vscode.git` as an extension dependency and reads unmerged stages directly through `git`.

## From source

Clone the repo and build the extension:

```bash
git clone https://github.com/Avni2000/MergeNB.git
cd MergeNB
npm install
npm run compile
npm run package
```

Load the built extension in VS Code:

- Press `F5` from the repo root to launch an Extension Development Host, or
- Package a VSIX with `npx vsce package` and install it like a release.

## Compatibility note

:::warning nbdime
MergeNB does not coexist with [`nbdime`](https://nbdime.readthedocs.io/) in the same merge flow.
If both are active, you can end up with merge artifacts that MergeNB cannot reliably parse.
Pick one notebook merge strategy per repo.
:::

The extension detects `merge.tool`, `merge.<name>.driver`, and `diff.<name>.driver` entries that look like
nbdime or a Jupyter tool, and will warn before opening a conflict.

## Invoking the resolver

Once installed, three entry points surface the command:

- Command Palette: **MergeNB: Find Notebooks with Merge Conflicts** (`merge-nb.findConflicts`)
- Keybinding: `Ctrl+Shift+M` / `Cmd+Shift+M` on an `.ipynb`
- Status bar: a badge appears when the workspace has unmerged notebooks

See [VS Code extension](./apps/vscode-extension) for the full command surface and
[Settings](./settings) for every configuration knob.
