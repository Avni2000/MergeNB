# Playwright

Lives in `packages/web/tests/`. Each spec boots a real web server, a temp-dir Git repo with a three-way merge conflict, and a Chromium page pointed at the resolver. The UI is exercised the same way a user would.

## Running

```bash
npm run test:pw                 # headless
npm run test:pw:headed          # watch the browser
npm run test:pw:ui              # Playwright's inspector UI
npm run test:pw:debug           # debug mode with pause
npx playwright test <file>.spec.ts --reporter=line
```

:::warning Always compile first
The server serves from `dist/web/`. If you change TypeScript under `packages/web/client` or `packages/core` and forget `npm run compile && npm run compile-tests`, your tests will exercise stale code.
:::

## Structure

| File | Purpose |
|---|---|
| `playwright.config.ts` (repo root) | Worker count, timeouts, global setup/teardown, reporters |
| `packages/web/tests/fixtures.ts` | Playwright Test fixtures: merge repo, session, page |
| `packages/web/tests/globalSetup.ts` | Starts the web server once per run |
| `packages/web/tests/globalTeardown.ts` | Stops it |
| `packages/web/tests/*.spec.ts` | Specs |

## Fixtures

`fixtures.ts` exposes a `test` with custom fixtures:

- Builds a temp Git repo from a `NotebookTriplet` (`base.ipynb`, `current.ipynb`, `incoming.ipynb`).
- Opens a conflict session against the running web server.
- Attaches a `page` already navigated to the session URL.
- Teardown deletes the temp repo and closes the browser.

Each spec declares what it wants:

```ts
test('cell reorder preserves unmatched cells', async ({ conflictSession }) => {
  // conflictSession.page is ready to drive
});
```

## Current spec coverage

- `editWarningOnBlur.spec.ts`: editing warnings
- `incomingNonConflictRegression.spec.ts`: non-conflict incoming cells still render
- `perCellResolution.spec.ts`: per-row choice behavior
- `rendermimeOutputs.spec.ts`: Jupyter mime output rendering
- `reorderUnmatch.spec.ts` and `reorderUnmatchApplyDisk.spec.ts`: move-invariant matching through apply
- `settingsMatrix.spec.ts`: every settings combination
- `syntaxHighlighting.spec.ts`: CodeMirror language detection per cell
- `takeAllButtons.spec.ts`: take-all flows
- `undoRedoActions.spec.ts`: history stack
- `unmatchGuardNoReorder.spec.ts`: reorder detection without false positives

## Isolated config per worker

`test-fixtures/shared/testRunnerShared.ts` provisions a per-worker `MERGENB_CONFIG_PATH` so parallel workers do not fight over the same settings file.
